Comparative positioning · 2026-05-12

Microsoft Agent Governance Toolkit + AAO-Governed.

Microsoft's open-source Agent Governance Toolkit (April 2026, MIT) is the runtime-enforcement layer for autonomous AI agents — a kernel-style policy engine at sub-millisecond latency. AAO-Governed (BGP · OBAC · AVS · HARP) is the cryptographic-governance layer on top. The right move is composability.

TL;DR

Microsoft shipped the right runtime layer. We're not building a competing one.

AAO-Governed solves three problems the MS toolkit explicitly does not address: (1) two agents proving they share a mandate without revealing the mandate, (2) every action signed and chained back to that mandate on a public append-only log, and (3) a halt-switch that any witness can pull, with a stake-bonded threshold counter-flip. These are not policy questions; they are cryptographic ones. They live above MS AGT's policy engine, not next to it.

L2 — Cryptographic governance (AAO-Governed)
BGP — mandate equality (Pedersen + Σ) · OBAC — attested action chain · AVS — replay verdicts · HARP — public halt quorum
L1 — Runtime enforcement (Microsoft Agent Governance Toolkit)
Agent OS · Agent Mesh · Agent Runtime · Agent SRE · Agent Compliance · Agent Marketplace · Agent Lightning
L0 — Agent frameworks
LangChain · CrewAI · AutoGen · Microsoft Agent Framework · Foundry Agent Service · LangGraph · LlamaIndex · PydanticAI · OpenAI Agents SDK

L1 stops bad actions before they happen. L2 makes the agent's commitments verifiable, the agent's history public, and the halt available to anyone who notices a violation. The two layers compose cleanly because L1 already exposes the right public interfaces — ToolCallInterceptor, BaseIntegration, PluginInterface, PolicyProviderInterface — for an L2 to plug in.

The composability layer (working demo)

We ship a small TypeScript wrapper that demonstrates exactly how the two layers compose. It is ~150 lines, MIT-licensed, and runnable. The wrapper intercepts MS AGT's ToolCallInterceptor path and adds two cryptographic gates around it:

  1. BGP gate (pre-call) — before any agent-to-agent tool call, run a Pedersen-commitment Σ-equality handshake on the two parties' mandates. If the mandates don't match, MS AGT never sees the call.
  2. OBAC attestation (post-decision) — every gated decision is appended to a public OBAC log entry signed by the agent's DID. The MS AGT decision payload is recorded verbatim alongside.
  3. HARP overlay (out-of-band) — a subscriber listens for HARP halt-quorum events. On flip, the wrapper triggers MS AGT's emergency kill switch via the documented runtime API.

Demo lives at demo/ms-composability/ in this repo. Run it with:

npm install @microsoft/agent-governance-sdk
npx tsx demo/ms-composability/index.ts

The wrapper is intentionally thin — most of the heavy lifting is delegated to MS AGT (policy enforcement) and to the AAO-Governed reference implementation at github.com/CrunchyJohnHaven/aiap (cryptographic primitives). The 150-line file is the composition, not the substrate.

Why composability, not competition

Three reasons we do not frame this as adversarial:

  • Different layers of the stack. Microsoft solved the runtime-enforcement problem. The cryptographic-governance problem is a different problem. Treating them as alternatives is a category error — like treating mTLS and OPA as competitors.
  • Procurement asymmetry favors composition. Enterprises will adopt MS AGT for reasons that have nothing to do with technical merit (procurement defaults, Azure integration, vendor consolidation). Building a competing runtime would force buyers into a choice they don't want to make. Building on top of MS AGT meets them where they already are.
  • Honest credit is durable. The MS toolkit's open-source posture is real (MIT, foundation trajectory, OWASP alignment, 9,500 tests). We have no interest in framing them as the enemy. Naming them as the runtime layer and us as the governance layer is the version of the story that survives the longest.

For the full positioning argument, see docs/POSITIONING_VS_MICROSOFT.md.

What we add — in one paragraph each

BGP — Bradley-Gavini Protocol

Two agents prove directive_A == directive_B without revealing the directive. Pedersen commitments on NIST P-256 plus a non-interactive Σ-protocol of equality (Schnorr + Fiat-Shamir). Sub-millisecond per handshake on commodity hardware. Used as a pre-flight check before MS AGT processes any inter-agent call: if the parties don't share a mandate, the call never enters the runtime.

OBAC — Origin-Bound Attestation Chain

An append-only public log binding each agent's actions to the mandate it committed to under BGP. Certificate-Transparency lineage (RFC 9162). Subjects can annotate their entries but cannot rewrite them. The point is to make agent history a public fact, not a vendor-controlled audit log.

AVS — Alignment-Verified Synthesizer

Independent third parties replay OBAC entries against the MS AGT policy state at the time, and publish verdicts. Community-Notes-style bridging synthesizer rather than a single source of truth. Misalignment becomes a public, dated, attested fact rather than an internal claim.

HARP — Halt-and-Rescue Protocol

Any witness who detects a verified violation can flip a stake-bonded halt bit. A FROST threshold of registered counter-witnesses can counter-flip. The wrapper subscribes to flip events and drives MS AGT's emergency kill switch automatically. Stake-bonding makes griefing economically irrational; threshold counter-flip makes single-actor griefing recoverable.

How to adopt both layers

For a team that has already adopted MS AGT (or is planning to):

  1. npm install @microsoft/agent-governance-sdk — the MS runtime layer. (Or pip install agent-governance-toolkit[full].)
  2. Clone github.com/CrunchyJohnHaven/aiap — the AAO-Governed reference implementation (TypeScript, 38/38 protocol tests).
  3. Drop in the wrapper from demo/ms-composability/. Three function exports: withBgpGate(), withObacAttestation(), subscribeHarpHalt().
  4. Register the wrapper with MS AGT via its ToolCallInterceptor hook. No core modifications to MS AGT required.

The integration is additive. If the wrapper is removed, MS AGT runs exactly as before. If MS AGT is swapped for a different runtime layer, the wrapper still functions against any runtime that exposes a comparable interceptor interface.
